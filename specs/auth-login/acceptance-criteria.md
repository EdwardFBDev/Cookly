# AUTH-Login Acceptance Criteria

- [ ] The app index route renders the Cookly login screen.
- [ ] The screen matches the provided Cookly visual direction: dark background, gold brand accent, centered welcome copy, elevated login card, and footer registration link.
- [ ] The screen includes email and password inputs with accessible labels.
- [ ] Password input is masked by default and can be shown or hidden.
- [ ] "Keep me logged in" can be toggled locally.
- [ ] Primary login, forgot password, register, Google, and Apple actions are visible.
- [ ] The implementation does not perform authentication, persistence, API calls, database calls, or navigation to unavailable routes.
- [ ] The implementation reuses existing theme tokens.
- [ ] The implementation does not add new dependencies.
